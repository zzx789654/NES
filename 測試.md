### 問題描述
用戶報告：弱點掃描頁面上傳資料後前端顯示 HTTP 500 錯誤，懷疑資料庫連線異常

### 根本原因分析
1. **Alembic Migration Schema 不完整**
   - `0001_initial_schema.py` vulnerabilities 表只定義 16 個欄位
   - 模型（models/scan.py）定義 31 個欄位（Nessus 完整支援）
   - 上傳時試圖插入不存在的欄位 → SQL OperationalError → HTTP 500

2. **測試環境衝突**
   - `main.py` 無條件執行 Alembic migration
   - 測試環境 SQLite 表已存在 → 重複執行失敗
   - `TESTING` 環境變數未被檢查

3. **Parser 精度不匹配**
   - EPSS: parser 四捨五入為 3 位小數，但 schema 期望 4 位
   - 日期: 只支援 ISO 格式，不支援 US 格式 (mm/dd/yyyy)
   - NaN: pandas NaN 未正確轉換為 JSON null

### 修復方案

#### 1. Migration 文件完整化 ✅
**修改**: `backend/alembic/versions/0001_initial_schema.py`
```python
# 舊：16 個欄位
# 新：31 個欄位（含所有 CVSS 版本、EPSS、VPR、Risk Factor、STIG Severity 等）
```

#### 2. Testing 環境隔離 ✅
**修改**: `backend/main.py` 中 `_run_migrations()`
```python
if os.getenv("TESTING", "").lower() == "true":
    Base.metadata.create_all(bind=engine)  # 使用 SQLAlchemy 初始化
    return
# 否則使用 Alembic
```

#### 3. Parser 邏輯改進 ✅
**修改**: `backend/services/nessus_parser.py`
- EPSS 精度改為 4 位小數
- 日期解析添加 `format='mixed'` 支援多格式
- NaN 轉換改為逐行檢查 `pd.isna()`
