## 一、資料持久化（最高優先）

### 1-1 上傳的 Nessus / CVE 掃描資料無法持久保存

- **現況**：上傳資料存於 React `useState`，重新整理即消失
- **影響**：無法累積歷史掃描、無法做跨版本比較
- **改善方向**：
  - 短期：存入 `localStorage`（限 5–10 MB）
  - 長期：FastAPI `/api/scans/upload` 接收 CSV → 解析 → 存入 PostgreSQL

```
✅ 已實作 API（backend/routers/scans.py）：
POST   /api/scans/upload     # 上傳 Nessus CSV / NVD JSON（含 EPSS 自動富化）
GET    /api/scans             # 取得所有掃描清單
GET    /api/scans/{id}        # 取得單筆掃描詳情
DELETE /api/scans/{id}        # 刪除掃描
GET    /api/scans/diff?base=&comp=  # 版本差異
```

### 1-2 NIST Audit 掃描記錄無法持久保存

- **現況**：同上，僅有兩筆 Demo 資料內建
- **改善方向**：
  - FastAPI `/api/nist/upload` → 存入 `audit_scans` / `audit_results` 表

```
✅ 已實作 API（backend/routers/nist.py）：
POST   /api/nist/upload       # 上傳 Audit CSV
GET    /api/nist/scans         # 取得所有 Audit 掃描
GET    /api/nist/scans/{id}    # 取得單筆結果
GET    /api/nist/diff?base=&comp=  # 版本差異
GET    /api/nist/trend         # 通過率趨勢資料
```

### 2-2 資料庫 Schema（PostgreSQL）

```sql
-- 弱點掃描
CREATE TABLE scans (
    id          SERIAL PRIMARY KEY,
    name        VARCHAR(200),
    source      VARCHAR(20),   -- 'nessus_csv' | 'nvd_json'
    scan_date   DATE,
    uploaded_at TIMESTAMPTZ DEFAULT NOW(),
    host_count  INT,
    vuln_count  INT
);

CREATE TABLE vulnerabilities (
    id          SERIAL PRIMARY KEY,
    scan_id     INT REFERENCES scans(id) ON DELETE CASCADE,
    plugin_id   VARCHAR(50),
    cve         VARCHAR(50),
    risk        VARCHAR(20),   -- Critical/High/Medium/Low/Info
    host        INET,
    port        VARCHAR(10),
    protocol    VARCHAR(10),
    name        TEXT,
    cvss        NUMERIC(4,1),
    epss        NUMERIC(6,4),
    vpr         NUMERIC(4,1),
    synopsis    TEXT,
    description TEXT,
    solution    TEXT,
    plugin_output TEXT
);

-- NIST Audit
CREATE TABLE audit_scans (
    id          SERIAL PRIMARY KEY,
    name        VARCHAR(200),
    scan_date   DATE,
    uploaded_at TIMESTAMPTZ DEFAULT NOW(),
    total       INT,
    passed      INT,
    failed      INT,
    warning     INT
);

CREATE TABLE audit_results (
    id          SERIAL PRIMARY KEY,
    scan_id     INT REFERENCES audit_scans(id) ON DELETE CASCADE,
    check_name  TEXT,
    status      VARCHAR(20),   -- PASSED/FAILED/WARNING
    description TEXT,
    policy_val  TEXT,
    actual_val  TEXT
);

-- IP 群組
CREATE TABLE ip_groups (
    id          SERIAL PRIMARY KEY,
    name        VARCHAR(100) UNIQUE,
    ips         JSONB,
    created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- 使用者
CREATE TABLE users (
    id          SERIAL PRIMARY KEY,
    username    VARCHAR(100) UNIQUE,
    hashed_pw   VARCHAR(255),
    role        VARCHAR(20) DEFAULT 'viewer',  -- admin/analyst/viewer
    created_at  TIMESTAMPTZ DEFAULT NOW()
);
```

## 三、部署設定（Ubuntu 24.04 / 22.04）✅ 已完成

### 3-1 Nginx 設定 ✅

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 靜態前端
    root /var/www/secvision;
    index index.html;
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 3-2 systemd 服務 ✅

```ini
[Unit]
Description=SecVision FastAPI Backend
After=network.target postgresql.service

[Service]
User=secvision
WorkingDirectory=/opt/secvision/backend
ExecStart=/opt/secvision/venv/bin/uvicorn main:app --host 127.0.0.1 --port 8000
Restart=always

[Install]
WantedBy=multi-user.target
```

### 3-3 安裝指令（快速部署）✅ → `deploy/install.sh`

```bash
# 1. 安裝依賴
sudo apt update && sudo apt install -y python3.11 python3.11-venv postgresql nginx

# 2. 建立資料庫
sudo -u postgres psql -c "CREATE USER secvision WITH PASSWORD 'changeme';"
sudo -u postgres psql -c "CREATE DATABASE secvision OWNER secvision;"

# 3. 安裝 Python 套件
python3.11 -m venv venv && source venv/bin/activate
pip install fastapi uvicorn sqlalchemy psycopg2-binary alembic python-jose passlib python-multipart pandas

# 4. 初始化資料庫
alembic upgrade head

# 5. 啟動服務
sudo systemctl enable --now secvision
sudo systemctl enable --now nginx
```
