### 2-2 資料庫 Schema（PostgreSQL）

```sql
-- 弱點掃描
CREATE TABLE scans (
    id          SERIAL PRIMARY KEY,
    name        VARCHAR(200),
    source      VARCHAR(20),   -- 'nessus_csv' | 'nvd_json'
    scan_date   DATE,
    uploaded_at TIMESTAMPTZ DEFAULT NOW(),
    host_count  INT,
    vuln_count  INT
);

CREATE TABLE vulnerabilities (
    id          SERIAL PRIMARY KEY,
    scan_id     INT REFERENCES scans(id) ON DELETE CASCADE,
    plugin_id   VARCHAR(50),
    cve         VARCHAR(50),
    risk        VARCHAR(20),   -- Critical/High/Medium/Low/Info
    host        INET,
    port        VARCHAR(10),
    protocol    VARCHAR(10),
    name        TEXT,
    cvss        NUMERIC(4,1),
    epss        NUMERIC(6,4),
    vpr         NUMERIC(4,1),
    synopsis    TEXT,
    description TEXT,
    solution    TEXT,
    plugin_output TEXT
);

-- NIST Audit
CREATE TABLE audit_scans (
    id          SERIAL PRIMARY KEY,
    name        VARCHAR(200),
    scan_date   DATE,
    uploaded_at TIMESTAMPTZ DEFAULT NOW(),
    total       INT,
    passed      INT,
    failed      INT,
    warning     INT
);

CREATE TABLE audit_results (
    id          SERIAL PRIMARY KEY,
    scan_id     INT REFERENCES audit_scans(id) ON DELETE CASCADE,
    check_name  TEXT,
    status      VARCHAR(20),   -- PASSED/FAILED/WARNING
    description TEXT,
    policy_val  TEXT,
    actual_val  TEXT
);

-- IP 群組
CREATE TABLE ip_groups (
    id          SERIAL PRIMARY KEY,
    name        VARCHAR(100) UNIQUE,
    ips         JSONB,
    created_at  TIMESTAMPTZ DEFAULT NOW()
);

-- 使用者
CREATE TABLE users (
    id          SERIAL PRIMARY KEY,
    username    VARCHAR(100) UNIQUE,
    hashed_pw   VARCHAR(255),
    role        VARCHAR(20) DEFAULT 'viewer',  -- admin/analyst/viewer
    created_at  TIMESTAMPTZ DEFAULT NOW()
);
```

## 三、部署設定（Ubuntu 24.04 / 22.04）

### 3-1 Nginx 設定

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 靜態前端
    root /var/www/secvision;
    index index.html;
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 代理
    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 3-2 systemd 服務

```ini
[Unit]
Description=SecVision FastAPI Backend
After=network.target postgresql.service

[Service]
User=secvision
WorkingDirectory=/opt/secvision/backend
ExecStart=/opt/secvision/venv/bin/uvicorn main:app --host 127.0.0.1 --port 8000
Restart=always

[Install]
WantedBy=multi-user.target
```

### 3-3 安裝指令（快速部署）

```bash
# 1. 安裝依賴
sudo apt update && sudo apt install -y python3.11 python3.11-venv postgresql nginx

# 2. 建立資料庫
sudo -u postgres psql -c "CREATE USER secvision WITH PASSWORD 'changeme';"
sudo -u postgres psql -c "CREATE DATABASE secvision OWNER secvision;"

# 3. 安裝 Python 套件
python3.11 -m venv venv && source venv/bin/activate
pip install fastapi uvicorn sqlalchemy psycopg2-binary alembic python-jose passlib python-multipart pandas

# 4. 初始化資料庫
alembic upgrade head

# 5. 啟動服務
sudo systemctl enable --now secvision
sudo systemctl enable --now nginx
```

### 9-1 🔴 登入端點暴力破解防護（Rate Limiting）

**弱點**：`POST /api/auth/token` 無限制失敗次數，可被用於憑證填充（Credential Stuffing）攻擊。

**修改計畫**：
- 新增 `slowapi` 套件至 `requirements.txt`
- 在 `main.py` 初始化 `Limiter`，設定預設限制
- 登入端點限制：**每 IP 每分鐘最多 10 次**
- 上傳端點限制：**每 IP 每分鐘最多 5 次**
- 回傳標準 HTTP 429 Too Many Requests

**修改檔案**：
- `backend/requirements.txt` — 新增 `slowapi==0.1.9`
- `backend/main.py` — 初始化 Limiter、加入 `SlowAPIMiddleware`
- `backend/routers/auth.py` — `@limiter.limit("10/minute")` 套用於 `/token`
- `backend/routers/scans.py` — `@limiter.limit("5/minute")` 套用於 `/upload`
- `backend/routers/nist.py` — `@limiter.limit("5/minute")` 套用於 `/upload`

**狀態**：🔴 待實作 → ✅ 已完成

---

### 9-2 🔴 密碼強度政策（Password Policy）

**弱點**：`/api/auth/register` 接受任意密碼（可為單一字元 "a"），無長度或複雜度要求。

**修改計畫**：
- 在 `schemas/auth.py` 的 `UserCreate` 新增 Pydantic validator：
  - 最短 **8 個字元**
  - 至少包含 **1 個大寫字母**
  - 至少包含 **1 個數字**
  - 至少包含 **1 個特殊字元**（`@$!%*?&`）
- 使用者名稱：僅允許英數字及底線，長度 3–50 字元

**修改檔案**：
- `backend/schemas/auth.py` — 新增 `@field_validator` 於 `password` 與 `username` 欄位

**狀態**：🔴 待實作 → ✅ 已完成

---

### 9-3 🔴 安全標頭中介軟體（Security Headers Middleware）

**弱點**：API 回應缺少標準安全標頭，易受 Clickjacking、MIME Sniffing、XSS 等攻擊。

**修改計畫**：
新增自訂 Starlette Middleware 於 `main.py`，在每個回應加入：

| 標頭 | 值 |
|------|----|
| `X-Content-Type-Options` | `nosniff` |
| `X-Frame-Options` | `DENY` |
| `X-XSS-Protection` | `1; mode=block` |
| `Referrer-Policy` | `strict-origin-when-cross-origin` |
| `Permissions-Policy` | `geolocation=(), camera=(), microphone=()` |
| `Content-Security-Policy` | `default-src 'self'` （API 限制） |

**修改檔案**：
- `backend/main.py` — 新增 `SecurityHeadersMiddleware` 類別

**狀態**：🔴 待實作 → ✅ 已完成

---

### 9-4 🔴 請求稽核日誌（Audit Logging Middleware）

**弱點**：無任何請求日誌或操作稽核記錄，無法追蹤誰何時做了什麼操作，違反 NIST AC-2、AU-2 控制項。

**修改計畫**：
新增 `AuditLogMiddleware` 記錄每個請求：
- 時間戳（UTC ISO 8601）
- 方法 + 路徑
- 客戶端 IP
- HTTP 狀態碼
- 回應耗時（ms）
- 格式：JSON 結構化日誌至 stdout，可接 ELK / CloudWatch

**修改檔案**：
- `backend/main.py` — 新增 `AuditLogMiddleware` 類別

**狀態**：🔴 待實作 → ✅ 已完成

---

### 9-5 🔴 檔案上傳安全加固（File Upload Hardening）

**弱點**：
1. 無檔案大小限制 → 超大 CSV 可造成記憶體耗盡（OOM）
2. 僅依副檔名判斷類型，未驗證檔案 Magic Bytes
3. CSV 解析無行數上限

**修改計畫**：
- 限制最大上傳大小：**50 MB**
- CSV 最大行數：**500,000 筆**
- 驗證 CSV 首字元（應為可見 ASCII），JSON 應以 `{` 或 `[` 開頭
- 超出限制回傳 HTTP 413

**修改檔案**：
- `backend/routers/scans.py` — 新增 `MAX_UPLOAD_SIZE = 50 * 1024 * 1024` 及驗證邏輯
- `backend/routers/nist.py` — 同上
- `backend/services/nessus_parser.py` — 新增 `max_rows` 參數保護

**狀態**：🔴 待實作 → ✅ 已完成

### 9-6 🟡 API 分頁支援防止 OOM（Pagination）

**弱點**：`GET /api/scans` 和 `GET /api/nist/scans` 回傳全部記錄，無分頁保護；大量資料可能造成記憶體耗盡或回應超時。

**修改計畫**：
- 新增可選查詢參數：`?page=1&page_size=50`（預設 page_size=100，最大 500）
- 若不傳則維持向後相容（回傳全部，但加記錄數上限警示）

**修改檔案**：
- `backend/routers/scans.py` — 新增 `page`, `page_size` 查詢參數
- `backend/routers/nist.py` — 同上

**狀態**：🟡 待實作 → ✅ 已完成

### 9-7 🟡 Nginx 資安強化（Nginx Security Hardening）

**弱點**：
1. 未加入任何 HTTP 安全標頭
2. 未限制請求體大小
3. Server header 洩漏版本資訊
4. 無請求速率限制

**修改計畫**：
在 `deploy/nginx.conf` 加入：
- `server_tokens off;` — 隱藏 Nginx 版本
- `client_max_body_size 50m;` — 與後端一致
- `add_header X-Frame-Options DENY;`
- `add_header X-Content-Type-Options nosniff;`
- `add_header Referrer-Policy strict-origin-when-cross-origin;`
- `limit_req_zone` + `limit_req` — 全域速率限制

**修改檔案**：
- `deploy/nginx.conf`

**狀態**：🟡 待實作 → ✅ 已完成

## 十、前端瀏覽器快取問題修復 🔴 進行中

### 根本原因

`index.html` 直接以固定 URL 載入本地 `.jsx` / `.js` 檔，Nginx 也未設定任何快取控制標頭。
瀏覽器依照 HTTP 預設快取策略（通常快取數天至數週），導致：
- 部署新程式碼後，瀏覽器仍讀取本地快取的舊版 JSX/JS
- 需要使用者手動 Ctrl+F5 強制重整才能看到新功能

### 修復計畫

| 修改 | 說明 |
|------|------|
| `deploy/nginx.conf` | 為 `index.html`、`.jsx`、`.js` 加入 `Cache-Control: no-cache` |
| `index.html` | script src 加入 `?v=__DEPLOY_VERSION__` 佔位符 |
| `deploy/redeploy-backend.sh` | 部署時以 git short hash 替換 `__DEPLOY_VERSION__` |

### 實作清單

- [x] `deploy/nginx.conf` — 新增 `location` 區塊停用應用程式檔快取
- [x] `index.html` — 加入版本查詢參數於本地 script 標籤
- [x] `deploy/redeploy-backend.sh` — 注入 git hash 到 index.html

**狀態**：✅ 已完成
